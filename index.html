Spare Hanger User Interface Guide

What's This?

This is a reference guide to the visual style and elements of Spare Hanger. Here you can find which components you should use for a particular feature on the site, or you can simply use it as an overall reference to the way things work.

All classes and styles defined and used on Spare Hanger (and on this guide) are namespaced under ui. This means that all elements without a class beginning with ui- will be uneffected by the styles defined here.

Grid System

Spare Hanger uses a grid system based on percentages. For instance, a block-level element such as 

<div class="ui-column-100">

would cover 100% of the width of its parent element. Typical usage of the grid system might look like this: 


<div class="ui-row">
	<div class="ui-column-30">
		This is a column that is 30% of the width
	</div>
	<div class="ui-column-70">
		This is a larger column this is 70% of the width
	</div>
</div>

Note that

<div class="ui-row">

is required. Grids may be nested as deeply as necessary.

Finally, any percentage may be used. For example, .column-43 will given a width of 43 percent.

Text Styles

Body text is set to a specific percentage of the page width. From there, all text sizes are measured in rem.

Spare Hanger is set in Proxima Nova.

Tooltips

This sentence has a link with a tooltip in it. 


<a href="" toolip="Yes, it does!">link with a tooltip</a>

Tooltips are used to tell a user some suplimentary information about what an element does. Tooltips should not be used unless completely necessary. If an element requires a tooltip, it generally means that the element is not appropriate or not well thought out.

When should I use a tooltip?

  • Any time you absolutely must tell a user something important about what an element does.

Popups are used for displaying extremely interuptive content. When clicked, the entire site is put at a standstill until the user interacts with the site. Therefore, popups should be used sparingly.

When should you use a popup?

  • Composing messages to other users
  • Confirming particularly destructive actions (deleting a brand)
  • Any time you want a user to be forced to interact with the site.

Alerts are useful for when you want to let a user know whether something happened or not, and also for letting users know about updates. Alerts will hide themselves when a user closes them, and take an optional temporary class to slowly fade off the page.

When should you use an alert?

  • There was an error saving something to the database
  • You want to let a user know that something was saved correctly
  • Any time you want a user to read something.

Spare Hanger provides a variety of ways to manipulate hover views on the page. In particular, you can add and filter out objects. You can use these methods from anywhere in the app. 


$(function() {
	// containerClass is optional for all methods, defaults to ".objects"

	// Filter objects by a single string, i.e. searching
	app.hoverviews.filterByString("red", containerClass);

	// Filter objects by multiple classes, i.e. sorting by categories
	app.hoverviews.filterByClasses([".red", ".green"], containerClass);

	// Directly adding objects to the container
	app.hoverviews.add($("<div class='hover-view'>...</div>"), containerClass);

	// Make an ajax call to add a number of items to the page
	app.hoverviews.ajax.getItems(amount, {category_one_id: 2}, imageWidth, sortingType, containerClass);
});

Option bars provide a way to present a user with multiple relevant choices to click based on the page they are currently on. Option bars should not be used for site-wide navigation, but for links to relevant pages. Options take any of the available button classes.

When should you use an option bar?

  • The top of a page to manage brands
  • Linking to different parts of a profile
  • Any time you want a way to display relevant links to the page you're on.

When should you use pills?

  • Switching between different objects
  • Linking to different parts of a profile
  • Any time you want to allow the user to switch the context of the page they're on.

Navbars are used as site-level navigation elements. They provide users information about the site and simple controls to access pages that should always be accessible within a single click. There should only be one navbar on the site.

When should I use a navbar?

  • When you want to have a way to display site-level navigation to the user.
