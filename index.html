Spare Hanger Style Guide

Grid

Spare Hanger uses a grid system based on percentages. For instance, a block-level element such as 

<div class="column-100">

would cover 100% of the width of its parent element. Typical usage of the grid system might look like this: 


<div class="row">
	<div class="column-30">
		This is a column that is 30% of the width
	</div>
	<div class="column-70">
		This is a larger column this is 70% of the width
	</div>
</div>

Note that

<div class="row">

is required and that the percentages in a .row must add up to 100%. Grids can also be nested.

Buttons


<a href="#" class="dangerous button">Button Text</a>

Use the standard .button for form actions and primary page actions. They should be used extensively around the site.

When should you use a button?

  • Form actions (Sign In, Create Item)
  • Primary page actions (Add Friend, Message)
  • Any time you want the user to click something.

Alerts


<div class="alert">
	<span class="ss-icon close">close</span>
	<strong>.alert:</strong> Check this out!
</div>

Alerts are useful for when you want to let a user know whether something happened or not, and also for letting users know about updates. Alerts will hide themselves when a user closes them.

When should you use an alert?

  • There was an error saving something to the database
  • You want to let a user know that something was saved correctly
  • Any time you want a user to read something.

close .alert: Check this out!
close .alert.warning: Watch yourself!
close .alert.error: Something went wrong!

Box


<div class="box">
	<h2>Box Heading</h2>
	<div class="box-inner">
		<p>Lorem ipsum dolor sit amet..</p>
	</div>
</div>

Boxes provide a way to display information that is seperate from the content around it. The header should be descriptive, but brief.

When should you use a box?

  • Admin panels
  • Settings pages
  • Any time your information is seperate from the content around it.

Hover Views


<div class="hover-view" style="width: 200px; height: 200px">
	<a href="" class="hover-img">
		<img src="#">
	</a>
	<a href="" class="hover-content">
		<h4>Object Name</h4>
		<p>Lorem ipsum dolor sit amet, consectetur adipisicing elit.</p>
	</a>
	<div class="hover-tools">
		<a href="" class="ss-icon">plus</a><span class="count">10</span> 
		<a href="" class="ss-icon">heart</a><span class="count">45</span> 
	</div>
</div>

Hover views are the main way that content gets displayed on Spare Hanger. An image is displayed which, when hovered over, reveals information about the object. Hover views work perfectly with Isotope.

Note that in order for a hover view to display correctly, its width and height must be the same as the width and height of the image.

When should you use a hover view?

  • General object display
  • Isotope-enabled layouts
  • Any time you want a visual look without much textual information.

Navigation

Option Bar


	<ul class="option-bar">
		<li><a href="" class="button"><i class="ss-icon">eye</i> Brands Followed</a></li>
		<li><a href="" class="button"><i class="ss-icon">key</i> Manage Brands</a></li>
		<li><a href="" class="button"><i class="ss-icon">plus</i> Create Brand</a></li>
		<li><a href="" class="dangerous button"><i class="ss-icon">delete</i> Delete Brand</a></li>
	</ul>

Option bars provide a way to present a user with multiple relevant choices to click based on the page they are currently on. Option bars should not be used for site-wide navigation, but for links to relevant pages. Options take any of the available button classes.

When should you use an option bar?

  • The top of a page to manage brands
  • Linking to different parts of a profile
  • Any time you want a way to display relevant links to the page you're on.


  • eye Brands Followed
  • key Manage Brands
  • plus Create Brand
  • delete Delete Brand
  • pencil Edit Brand
  • compose Add a Product

Pills


<ul class="pills">
	<li class="active"><a href="">Items</a></li>
	<li><a href="">Outfits</a></li>
	<li><a href="">Brands</a></li>
</ul>

Pills are a sub-nav that is used for switching contexts. Pills shouldn't really be used for navigating to other parts of the site. Instead, they should swith the information currently being displayed.

When should you use pills?

  • Switching between different objects
  • Linking to different parts of a profile
  • Any time you want to allow the user to switch the context of the page they're on.

  • Items
  • Outfits
  • Brands
